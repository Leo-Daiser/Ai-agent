Multi-Agent Interview Coach

Проект представляет собой консольную систему для проведения технического интервью с использованием многоагентной архитектуры.
Система разделяет процесс интервьюирования и оценки ответов на независимые роли, что позволяет сохранить чистую логику, расширяемость и управляемость поведения.

Назначение проекта

Проект предназначен для:

симуляции технического интервью,

оценки качества ответов кандидата,

выявления подтверждённых навыков и пробелов в знаниях,

формирования структурированного финального отчёта.

Система ориентирована на образовательные и исследовательские сценарии, а не на автоматическое принятие кадровых решений.

Архитектура

В основе проекта лежит принцип жёсткого разделения ролей.
Агенты не вызывают друг друга напрямую и не подменяют ответственность соседних компонентов.

Основные роли

Интервьюер (InterviewerAgent)
Отвечает исключительно за диалог:

формулирует и задаёт вопросы,

поддерживает ход интервью,

реагирует на некорректные или уточняющие ответы пользователя.

Интервьюер не оценивает ответы и не делает выводов о кандидате.

Наблюдатель (ObserverAgent)
Отвечает за анализ:

оценивает ответы кандидата,

фиксирует сильные стороны и пробелы,

рекомендует направление следующего вопроса,

формирует финальный аналитический отчёт.

Наблюдатель не взаимодействует напрямую с пользователем.

Координация

Состояние интервью хранится в общей структуре сессии.
Агенты обмениваются данными только через неё, что:

упрощает отладку,

исключает скрытые зависимости,

позволяет масштабировать систему новыми ролями.

Логика работы

Инициализируется сессия интервью.

Интервьюер запрашивает у модели следующий вопрос.

Пользователь вводит ответ.

Ответ анализируется наблюдателем.

Наблюдатель:

обновляет профиль кандидата,

формирует рекомендации по продолжению интервью.

Интервью продолжается до достижения условий завершения.

Генерируется финальный отчёт.

Финальный отчёт

Отчёт формируется наблюдателем и включает:

подтверждённые навыки,

пробелы в знаниях,

качественные комментарии по ответам,

персональный roadmap с конкретными рекомендациями.

Отчёт не содержит дублирующихся пунктов и не использует абстрактные формулировки без практического смысла.

Ключевые принципы

Разделение ответственности

Отсутствие жёстко заданного банка вопросов

Контекстная генерация вопросов

Анализ на основе всей истории интервью, а не отдельных ответов

Возможность расширения без переписывания ядра системы

Ограничения

Проект не предназначен для автоматической фильтрации кандидатов.

Оценка носит рекомендательный и аналитический характер.

Качество работы зависит от корректности входных ответов и контекста.

Возможности расширения

Архитектура допускает:

добавление новых ролей (например, технический эксперт или риск-оценщик),

внедрение формальных моделей оценки,

логирование и экспорт результатов,

интеграцию с внешними интерфейсами.
